Three Questions Every Summary Should Answer

QuestionWhat It MeasuresTools
What is typical?Central tendencymean, median, mode
How spread out?VariabilitySD, variance, IQR, range
What shape?Distributionskewness, kurtosis, histograms
💡 AI vs. Your Judgment

AI computes summaries instantly. You decide which summary is honest for your data, and you explain what it means in context. A mean wage of $42 means something very different depending on whether the data is symmetric or heavily skewed by a few executives.

Measures of Central Tendency

Mean

The arithmetic average. Best when data is symmetric with no extreme outliers.

salaries <- c(40000, 40000, 65000, 90000, 145000, 150000, 550000)
mean(salaries)
# [1] 154285.7  — pulled high by the outlier

# With NAs present, use na.rm
mean(salaries2, na.rm = TRUE)

Median

The middle value when sorted. Not affected by outliers — use when data is skewed.

median(salaries)
# [1] 90000  — much more representative of "typical"

# If mean ≠ median → likely skewed with outliers

Mode

The most frequently occurring value. Useful for categorical data.

# R has no built-in mode function for this purpose
# Use sort(table()) to find most frequent values
sort(table(salaries), decreasing = TRUE)[1:5]

# If highest frequency is 1 → no mode

When to Use Which

MeasureUse WhenWeakness
MeanData is symmetric, no extreme outliersPulled by outliers
MedianData is skewed or outliers presentIgnores magnitude of values
ModeCategorical data, or need most common valueMay not exist or be unique

Measures of Spread

Spread Relative to the Mean

# Variance and Standard Deviation
var(salaries)    # variance (squared units)
sd(salaries)     # standard deviation (same units as data)

Spread Relative to the Median

# Range: max - min
max(data$Spending) - min(data$Spending)
# or
diff(range(data$Spending))

# IQR: middle 50% of data (Q3 - Q1)
IQR(data$Spending, na.rm = TRUE)

Skewness

Measures how asymmetric a distribution is. Use the skew() function from the semTools package.

library(semTools)
skew(salaries)
# Output: skew (g1), se, z, p
# Evaluate the z-value against thresholds:

Z-Score Thresholds for Skewness & Kurtosis

Sample SizeProblematic z Range
n < 50 (small)Outside −2 to 2
50 ≤ n ≤ 300 (medium)Outside −3.29 to 3.29
n > 300 (large)Outside −7 to 7 (prefer visuals)

Interpreting Direction

  • Positive skew (right-skewed): longer tail to the right, mean > median
  • Negative skew (left-skewed): longer tail to the left, mean < median
  • Zero skew: symmetric distribution, mean ≈ median

Kurtosis

Measures how many observations are in the tails. The kurtosis() function from semTools subtracts 3, so compare to 0.

kurtosis(salaries)
# Output: Excess Kur (g2), se, z, p
# Same z-thresholds as skewness apply
TypeExcess KurtosisMeaning
Mesokurtic≈ 0Normal tails
Leptokurtic> 0 (significant)Heavy tails, sharp peak
Platykurtic< 0 (significant)Light tails, flat peak

Visualizing Quantitative Data

Histogram

Shows distribution shape by grouping values into bins. The key parameter is binwidth.

ggplot(data, aes(x = salary_usd)) +
  geom_histogram(binwidth = 15000, fill = "steelblue", color = "white") +
  labs(title = "Distribution of Salary",
       x = "Salary (USD)", y = "Count")

Density Plot

A smoothed histogram. Add a mean line with geom_vline().

ggplot(data, aes(x = salary_usd)) +
  geom_density(fill = "lavender", color = "purple", alpha = 0.6) +
  geom_vline(aes(xintercept = mean(salary_usd, na.rm = TRUE)),
             color = "red", linetype = "dashed") +
  labs(title = "Salary Density", x = "Salary", y = "Density")

Boxplot

Summarizes the five-number summary and flags outliers (points beyond 1.5 × IQR).

# Single variable
ggplot(data, aes(x = salary_usd)) +
  geom_boxplot(fill = "lightblue", color = "navy")

# Grouped by category
ggplot(data, aes(x = salary_usd, fill = company_size)) +
  geom_boxplot() +
  labs(title = "Salary by Company Size")

Detecting Outliers Numerically

Q1 <- quantile(data$salary_usd, 0.25)
Q3 <- quantile(data$salary_usd, 0.75)
IQR_val <- IQR(data$salary_usd)

LowerBound <- Q1 - 1.5 * IQR_val
UpperBound <- Q3 + 1.5 * IQR_val

# Any value outside these bounds is a potential outlier

Summarizing Categorical Data

Frequency Tables

# Frequencies
freq <- table(data$company_size)

# Relative frequencies (proportions)
prop <- prop.table(freq)

# Cumulative frequencies
cumfreq <- cumsum(freq)

# Combine into a table
rbind(freq, prop, cumfreq)

Bar Chart

ggplot(data, aes(x = company_size, fill = company_size)) +
  geom_bar(show.legend = FALSE) +
  labs(title = "Job Postings by Company Size",
       x = "Company Size", y = "Count") +
  scale_fill_manual(values = c("tomato", "steelblue", "seagreen"))

Key Takeaways

  • Mean vs. Median: If they differ substantially, the data is skewed — report the median
  • Always visualize: Numbers alone can mislead; histograms and boxplots confirm the story
  • Skewness z-scores: Use sample-size thresholds, not arbitrary cutoffs
  • Outliers are not errors: Investigate before removing
  • Categorical data: Use table() + prop.table(), visualize with bar charts
  • ggplot2 layers: Build charts incrementally with +
📐 The Descriptive Stats Workflow

Load data → Check types → Central tendency → Spread → Shape (skew/kurtosis) → Visualize → Confirm numbers match visuals
