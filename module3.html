Overview

Data preparation is the process of inspecting, cleaning, and reshaping a dataset so it is ready for analysis. This is where most analytical errors are introduced — a wrong data type or undetected missing value will silently corrupt every result downstream.

The workflow: Inspect first → Clean deliberately → Verify visually. Always run summary() before touching anything else.

Data Types and Coercion

R assigns a type to every variable when a dataset is loaded. The wrong type causes silent errors — a number stored as character breaks mean() without warning.

Loading Data with Correct Types

library(tidyverse)
data <- read.csv("data/myfile.csv", stringsAsFactors = TRUE)

The stringsAsFactors = TRUE argument converts text columns to factors automatically — saving manual as.factor() calls and making summary() output useful.

Coercion Pitfall

Applying as.numeric() directly to a factor returns internal level codes (1, 2, 3...), not the actual values. Fix: as.numeric(as.character(x))

The dplyr Toolkit

dplyr provides functions that each do one thing cleanly. The power comes from chaining them together.

VerbPurposeExample
filter()Keep rows matching a conditionfilter(data, Age > 30)
select()Keep specific columnsselect(data, Name, Age)
arrange()Sort rowsarrange(data, desc(Score))
mutate()Create/modify columnsmutate(data, BMI = Weight/Height^2)
summarize()Collapse to summary statssummarize(data, avg = mean(Score))
group_by()Set up groups for summarizegroup_by(data, Region)

filter() — Subset Rows

# Single condition
adults <- filter(data, Age >= 18)

# AND (both must be true)
result <- filter(data, Age >= 18 & Income > 50000)

# OR (either can be true)
result <- filter(data, Region == "East" | Region == "West")

select() — Choose Columns

small <- select(data, Name, Age, Income)

arrange() — Sort

# Ascending (default)
sorted <- arrange(data, Age)

# Descending
sorted <- arrange(data, desc(Income))
Key distinction: sort() works on vectors. arrange() works on data frames.

mutate() — Create New Columns

# Calculated column
data <- mutate(data, Profit = Revenue - Cost)

# Binary category with ifelse
data <- mutate(data, Category = ifelse(Score > 80, "High", "Low"))

# Multiple categories with case_when
data <- data %>% mutate(Tier = case_when(
  Score > 90 ~ "A",
  Score > 70 ~ "B",
  Score > 50 ~ "C",
  TRUE ~ "D"
))

The Pipe Operator (%>%)

The pipe takes the output of the left side and passes it as the first argument to the right side. It turns nested function calls into readable top-to-bottom pipelines.

# Without pipe (hard to read)
head(arrange(select(filter(data, Age > 30), Name, Income), desc(Income)))

# With pipe (reads like a sentence)
data %>%
  filter(Age > 30) %>%
  select(Name, Income) %>%
  arrange(desc(Income)) %>%
  head()

Handling Missing Data

Missing values (NA) are common and must be handled deliberately. The choice depends on why data is missing.

Three Strategies

  1. Omission — Remove rows with NAs. Risk: reduces sample size, may introduce bias.
  2. Imputation — Replace NAs with a reasonable value (mean for numeric, mode for categorical). Risk: distorts relationships if many values are missing.
  3. Ignore — Use na.rm = TRUE in functions that support it. Simplest when few NAs exist.

Key Functions

# Detect NAs
sum(is.na(data))              # Count all NAs in dataset
sum(is.na(data$column))       # Count NAs in one column

# Ignore NAs in calculations
mean(data$Score, na.rm = TRUE)

# Remove rows with NAs
clean <- na.omit(data)              # Base R: removes if ANY column has NA
clean <- data %>% drop_na()        # Tidyverse: same behavior
clean <- data %>% drop_na(Score)   # Tidyverse: only where Score is NA

# Convert specific values to NA
data <- data %>% mutate(Job = na_if(Job, "Unknown"))
Critical insight: If NAs are concentrated in a specific group (e.g., one region or demographic), removing them introduces selection bias. Always check where your NAs are before deciding how to handle them.

Group By and Summarize

The group_by() + summarize() pattern is one of the most frequently used in real analysis — comparing statistics across groups is the starting point for almost every business question.

# Mean income by region
data %>%
  group_by(Region) %>%
  summarize(
    mean_income = mean(Income, na.rm = TRUE),
    median_income = median(Income, na.rm = TRUE),
    count = n()
  )

group_by() doesn't compute anything — it sets up the data so summarize() runs within each group. The result has one row per group.

Counting

# Count observations matching a condition
data %>% filter(Status == "Active") %>% count()

# Count per group
data %>% group_by(Region) %>% count()

Multi-Variable Visualization

Visualization serves as a quality check on your data preparation — if the chart looks wrong, the cleaning has a problem.

Scatterplot — Two Continuous Variables

ggplot(data, aes(x = Age, y = Income)) +
  geom_point() +
  geom_smooth(method = "lm", color = "red") +
  labs(title = "Age vs Income", x = "Age", y = "Income ($)")

Goal: characterize the relationship — positive, negative, or nonexistent. Add color = GroupVar inside aes() to see patterns by group.

Grouped Bar Chart — Two Categorical Variables

ggplot(data, aes(x = Region, fill = Status)) +
  geom_bar(position = "dodge") +
  labs(title = "Status by Region", x = "Region", y = "Count")

position = "dodge" places bars side by side. Without it, bars stack.

Boxplot Across Groups

ggplot(data, aes(x = Region, y = Income, fill = Region)) +
  geom_boxplot() +
  theme_minimal()

Use when comparing a continuous variable's distribution across categories.

Practice Problems

  1. Load a dataset with stringsAsFactors = TRUE. Run summary(). Identify which variables are factors and which are numeric.
  2. Use filter() with a compound condition (AND) to subset your data. Print the row count with nrow().
  3. Use mutate() with ifelse() to create a binary category based on a numeric threshold. Use table() to count each category.
  4. Use group_by() + summarize() to calculate the mean of a numeric variable for each level of a categorical variable.
  5. Create a scatterplot with geom_point() and add a trendline with geom_smooth(method = "lm"). Describe the relationship.
  6. Create a grouped bar chart with position = "dodge". Which group has the highest count?

Key Takeaways

  • Always check types first — summary() + class() before any analysis
  • Each dplyr verb does one thing — power comes from chaining with %>%
  • Missing data requires judgment — omission, imputation, or ignore depends on context
  • group_by() + summarize() answers business questions — comparing means across groups
  • Visualization verifies your cleaning — if the chart looks wrong, the data prep has a problem
