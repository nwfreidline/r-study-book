What is Statistics?

Statistics is the methodology of extracting useful information from a data set. Numerical results must be accompanied with clearly stated actionable business insights.

Two Main Branches

BranchPurpose
DescriptiveCollecting, organizing, and presenting data
InferentialDrawing conclusions about a population based on sample data
💡 Key Insight

AI dominates descriptive statistics (automation of summaries and visualizations). Human judgment remains essential for inferential statistics — defining populations, assessing representativeness, and interpreting results in context.

Vectors

A vector is the simplest data structure in R — a set of elements of the same type.

# Four ways to create vectors
c(1, 2, 3, 4, 5)          # combine values
1:10                        # integer sequence
seq(0, 30, 5)              # custom sequence: 0, 5, 10, ..., 30
rep(1:3, times = 4)        # repeat: 1 2 3 1 2 3 1 2 3 1 2 3

x <- c(1, 2, 3, 100, -10, 40)
sort(x)                     # ascending
sort(x, decreasing = TRUE) # descending
mean(x)                     # average
max(x) - min(x)             # range

Data Frames

A data frame is a table where rows are observations and columns are variables. Each column can hold a different data type.

state <- c("Alaska", "Arizona", "Arkansas")
year <- c(1998, 2010, 2016)
limit <- c(1, 2.5, 3)

df <- data.frame(state, year, limit)
ncol(df)   # 3 columns
nrow(df)   # 3 rows
dim(df)    # 3 3

Loading Data

# Base R
gig <- read.csv("data/gig.csv")

# Tidyverse (faster, better defaults)
library(tidyverse)
gig <- read_csv("data/gig.csv")

# Always use RELATIVE paths
# Good:  "data/file.csv"
# Bad:   "C:/Users/name/Documents/data/file.csv"
⚠️ Relative vs Absolute Paths

Always use relative paths ("data/file.csv") instead of absolute paths. Absolute paths break when you share code or move to a different machine. Never use setwd() in .Rmd files.

Missing Data

StrategyWhen to UseR Function
OmitFew NAs, missingness is randomna.omit(), drop_na()
ImputeMany NAs, can't lose rowsReplace with mean/median
IgnoreFunction supports itna.rm = TRUE
# Count NAs
sum(is.na(gig))              # total in dataset
sum(is.na(gig$Wage))         # in one variable

# Remove rows with NAs
clean <- na.omit(gig)         # base R
clean <- gig %>% drop_na()   # tidyverse

# Ignore NAs in calculations
mean(gig$Wage, na.rm = TRUE)
💡 The Data Loading Workflow

Load → Check types (str) → Fix types → Handle NAs → Analyze
This sequence applies to every dataset you'll ever work with. Build it as muscle memory.

Key Takeaways

  • R is learned by doing — type every example, don't just read
  • Silent errors are the real danger — code runs but gives wrong output
  • Always check types after loading — wrong types cause wrong results
  • Use relative paths — makes code portable and shareable
  • AI writes code; you verify it — understand what every line does
